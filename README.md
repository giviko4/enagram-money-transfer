# TRANSACTION SIMULATION

### პროექტის მიმოხილვა
ეს არის სრულფასოვანი Full Stack აპლიკაცია, რომელიც შექმნილია **Enagram**-ის ტექნიკური დავალების ფარგლებში. პროექტი ახდენს ფულის გადარიცხვის სიმულაციას ვირტუალურ ანგარიშებს შორის.

---

🚀 გაშვების ინსტრუქცია (Docker)
პროექტი სრულად კონტეინერიზებულია, რაც უზრუნველყოფს მის მყისიერ გაშვებას ნებისმიერ გარემოში დამატებითი კონფიგურაციის გარეშე.
დარწმუნდით, რომ Docker Desktop ჩართულია.
გარემო ცვლადების მომზადება (.env):
უსაფრთხოების სტანდარტების დასაცავად, .env ფაილი არ არის ატვირთული რეპოზიტორში. პროექტის გასაშვებად:
მნიშვნელოვანი: ფაილი უნდა შეიქმნას ზუსტად იმ საქაღალდეში, სადაც მდებარეობს docker-compose.yml ფაილი.

ჩაწერეთ .env ფაილში შემდეგი მონაცემები:
DB_PASSWORD= YourStrongPassword123! |
DB_PORT=1599

(შენიშვნა: გამოყენებულია პორტი 1599, რათა თავიდან იქნას აცილებული Windows-ის სისტემური პორტების კონფლიქტი).
სწორი საქაღალდის არჩევა (კრიტიკულია ⚠️):
თუ პროექტს ZIP ფაილად გადმოწერთ, ტერმინალში აუცილებლად უნდა შეხვიდეთ უშუალოდ იმ საქაღალდეში, სადაც docker-compose.yml ფაილი დევს ანუ VsCode ტერმინალში დაწერე:
cd enagram-money-transfer-main
სისტემის დაქოქვა:
გაუშვით შემდეგი ბრძანება:
docker-compose up --build
აპლიკაციის ინტერფეისი:
მას შემდეგ, რაც ტერმინალი დაწერს, რომ კონტეინერები მზად არის, გადადით ბმულზე:
👉 http://localhost:3000



🛠️ პრობლემების აღმოფხვრა (Windows-ის მომხმარებლებისთვის)
თუ ბრძანების გაშვებისას მიიღებთ შეცდომას ports are not available ან access forbidden, ეს ნიშნავს, რომ Windows-ის WinNAT სერვისს პორტები რეზერვირებული აქვს.
გამოსავალი: გახსენით PowerShell ადმინისტრატორის უფლებით და გაუშვით:
Powershell
net stop winnat
net start winnat
ამის შემდეგ თავიდან გაუშვით docker-compose up.




🗄️ მონაცემთა ბაზის შემოწმება (SSMS)
მონაცემების (ანგარიშების ბალანსების და ტრანზაქციების ისტორიის) ბაზის დონეზე გადასამოწმებლად გამოიყენეთ SQL Server Management Studio (SSMS):
1. დაკავშირება (Connection):
Server Name: localhost,1599 |
(მნიშვნელოვანია: პორტის წინ გამოიყენეთ მძიმე). |
Authentication: SQL Server Authentication |
Login (User Name): sa |
Password: YourStrongPassword123! |
SSL/Encryption: დააჭირეთ ღილაკს Options >>, გადადით Connection Properties ჩანართში და აუცილებლად მონიშნეთ Trust server certificate. |
2. მონაცემების ნახვა (Viewing Data):
მას შემდეგ, რაც სერვერს დაუკავშირდებით, მარცხენა მენიუში (Object Explorer) მიჰყევით ამ ნაბიჯებს:
გახსენით საქაღალდე Databases > PartnerDb.
გახსენით საქაღალდე Tables.
ანგარიშების სანახავად: დააწკაპუნეთ მაუსის მარჯვენა ღილაკით dbo.Accounts-ზე და აირჩიეთ Select Top 1000 Rows. აქ დაინახავთ Alice-ს, Bob-ს და Charlie-ს მიმდინარე ბალანსებს.
ტრანზაქციების სანახავად: დააწკაპუნეთ მაუსის მარჯვენა ღილაკით dbo.Transactions-ზე და აირჩიეთ Select Top 1000 Rows. აქ გამოჩნდება შესრულებული გადარიცხვების ისტორია.




🛠️ ტექნიკური მახასიათებლები
Backend (.NET 8): გამოყენებულია Minimal API არქიტექტურა და EF Core Transactions. გადარიცხვის ოპერაცია არის ატომური — თანხის ჩამოჭრა და დამატება ხდება ერთიანი Transaction Scope-ის ქვეშ, რაც გამორიცხავს მონაცემთა დაკარგვას.
Database (MSSQL 2022): მონაცემთა ბაზა ავტომატურად ინიციალიზდება (Seeding) პირველივე გაშვებისას სატესტო ანგარიშებით (Alice, Bob, Charlie).
Frontend (React + Vite): ინტერფეისი აგებულია Tailwind CSS-ის გამოყენებით. დანერგილია Surgical UI Updates, რაც უზრუნველყოფს ბალანსების მყისიერ ასახვას გადარიცხვის შემდეგ.
Docker Orchestration: სამივე სერვისი (DB, API, Web) დაკავშირებულია შიდა იზოლირებული ქსელით.




🤖 მუშაობის პროცესი (Gemini AI)
პროექტი მომზადდა AI-Assisted Development მეთოდოლოგიით, Gemini-ს აქტიური გამოყენებით.
როგორ გამოვიყენე AI:
არქიტექტურის დაგეგმვა: Gemini-ს დახმარებით მოხდა .NET ბექენდისა და MSSQL სქემის ოპტიმალური სტრუქტურირება.
პრობლემების გადაჭრა: AI ინსტრუმენტები გამოვიყენე Windows-ის სპეციფიკური ქსელური ხარვეზების (WinNAT პორტების რეზერვაცია) დიაგნოსტიკისა და ოპერატიული აღმოფხვრისთვის.
დოკერიზაცია: Docker Compose-ის კონფიგურაცია და Healthcheck-ების გამართვა მოხდა ზუსტი პრომპტინგის საშუალებით, რათა მიგვეღო "Single Command Launch" გამოცდილება.
